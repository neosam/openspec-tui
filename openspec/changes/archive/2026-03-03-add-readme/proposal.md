## Why

The project has no README.md. New users or contributors have no way to understand what openspec-tui is, what it does, or how to build and run it. A README provides the essential entry point for anyone discovering the project.

## What Changes

- Add a `README.md` to the project root with:
  - Project description (what openspec-tui is and what it does)
  - Prerequisites (Rust toolchain, openspec CLI)
  - Build and run instructions (cargo, Nix flake dev shell)
  - Usage overview (keyboard navigation, screen flow)
  - License information if applicable
